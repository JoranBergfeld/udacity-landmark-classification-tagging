Question: Outline the steps you took to get to your final CNN architecture and your reasoning at each step. Describe why you think the architecture is suitable for the current problem.

Answer:

Honestly, the scripts did not give me much flexibility. The lack of modern GPU support had me try various low-training time configurations, but in the end I decided against it. I couldn't push any config beyond 80% with the low-training time configs.

This was all rooted that the provided requirements.txt defined old torch and torchvision dependencies, which did not recognise my GPU. I eventually upgraded them to a more recent version, which allowed me to leverage the GPU.

Once I got the GPU working, I went straight for the convnext_tiny architecture. That is because it came out as the best architecture in my previous experimentation runs. View the in-depth analysis on the experiments I ran at the analysis markdown file. Considering all the backbone parameters remain frozen, it's a simple manner of picking the right architecture and training the Linear head for the 50 classes that we will feed it. The approach that ImageNet teaches a modern convolutional backbone is suitable for landmarks. Edges and textures in the early layers, object parts and the whole scene/building structure in the deeper layer. The landmark set only has about 5000 images, which means training the backbone myself will severely overfit the model.

Now play with the hyperparameters and see which performance you can get on the validation set. You should get at least 60% for a passing grade, but a good model choice and a good training strategy could get you up to 80% or so. Let's see how close you can get!

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
